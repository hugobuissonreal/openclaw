Vue globale des agents

Qui pilote, qui supervise, qui exécute.

Cette page montre la hiérarchie agentique du système Content OS. L’objectif n’est pas seulement de voir des noms, mais de comprendre comment l’information circule entre le global, le CEO, les managers éventuels et les workers spécialisés.

CEO = orchestration Manager = supervision Worker = exécution Global = cadre partagé

Principe

Le global aligne tout le mondeBusiness, audience, stratégie, règles système, standards partagés.
Le CEO orchestreIl arbitre, délègue, consolide et décide des grandes évolutions.
Le manager absorbe la complexitéUtile seulement quand plusieurs workers doivent être structurés.
Le worker produit et apprendIl exécute sur un format précis, mesure, analyse et améliore.

Ce que le CEO voit

GlobalContexte, système, mémoire, standards.
AnalyticsSynthèses hebdo, scoreboard, tendances.
OpsProtocoles, architecture, registre d’agents.
CEO privéOrchestration, roadmap, learnings système.

Ce que le worker voit

Global utileBusiness, audience, offres, stratégie.
SharedValidation, assets, workflows communs.
Worker privéMission, format, pipeline, metrics.
Protocoles ciblésÉcriture Supabase, lecture ciblée si besoin.

Flux d’information

1. Le global descendLe cadre métier et système est partagé avec tous les niveaux.
2. Le worker remonteRésultats, analyses, learnings locaux, signaux de performance.
3. Le CEO consolideIl transforme les signaux locaux en décisions plus globales.
4. Le manager filtreSi présent, il sert de tampon entre pilotage global et exécution.

Décision structurelle

Le manager n’est pas obligatoire au départ. Dans une petite architecture, on peut fonctionner avec :

CEO → WorkerQuand il y a peu d’agents et peu de complexité.
CEO → Manager → WorkersQuand le nombre de formats, de plateformes ou de workers commence à exploser.
